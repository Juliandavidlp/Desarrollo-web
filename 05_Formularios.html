Formularios

Un formulario HTML es una sección de un documento que contiene texto, código, elementos especiales llamados controles:
casillas de verificación (checkboxes), botones de opción (radio buttons), menús, etc.; y rótulos (labels) en esos controles. Los usuarios normalmente “completan” un formulario
modificando los valores de sus controles (introduciendo texto, seleccionando objetos de un menú, etc.), antes de enviar el formulario a un agente para que lo procese (por ej.: a un servidor web o a un servidor de correo).

La etiqueta < form>< / form>

Se utiliza la etiqueta contenedora < form >, que incluye todos los campos necesarios para su funcionamiento. Hay tipos de campos adecuados para cada tipo de dato.
Existe un elemento especial, el botón, que puede enviar los datos del formulario al servidor o limpiar el contenido de sus campos.

Atributos action y method

    La etiqueta < form> admite diversos atributos. Entre ellos:

  • action: define el tipo de acción que se llevará a cabo (enviar a un mail o procesar su contenido con un script). Aquí debemos indicar la ruta a la que se enviará la petición HTTP con la información del formulario.
  • method: indica si la petición HTTP será GET o POST. Ojo, a este atributo hay que agregarlo.
  • enctype: se usa cuando enviamos el formulario por correo y lleva el valor "text/plain" para que el formulario se envíe en forma de texto plano.

La etiqueta Label

< label>: se usa para especificar el nombre de un campo -un elemento- dentro de un formulario, y es información para el usuario.
Admite el atributo for, que debe tener el mismo valor que el atributo id del campo asociado (input, select o textarea).

El Input

    Atributos de la etiqueta < input> :

  • type: este atributo puede tener distintos valores o posibilidades: text, email, checkbox, color, date, file, hidden, etc. en función del tipo de campo que queremos que ingrese el usuario.
  • id: este atributo es obligatorio si en el elemento label hay un atributo for, en tal caso el input debe coincidir con el elemento al que se refiere y contener un identificador único en la página. Hay que agregarlo
  • name: representa el nombre asignado al campo cuando se envía la petición HTTP.
  • size: define el tamaño de la caja de texto en número de caracteres visibles.
  • maxlength: indica el tamaño máximo del texto, en número de caracteres, que puede ser escrito en el campo.
  • value: representa el valor que se le asignará al campo cuando se envíe la petición HTTP. Permite asignarle un valor por defecto al campo.

El atributo type

    El atributo type de la etiqueta < input > valida el tipo de dato de entrada que ingresa el usuario, tiene distintos valores posibles:

  • text: permite ingresar una cadena alfanumérica
  • number: permite validar un número ingresado por el usuario
  • date: ofrece un calendario para cargar una fecha
  • datetime-local: ofrece otro calendario, permite cargar una fecha y una hora
  • password: oculta el texto que ingresamos.
  • email: valida que el usuario ingrese un correo de manera nativa, un correo electrónico válido
  • url: valida que el usuario ingrese una URL
  • image: define una imagen como botón de envío
  • tel: define un campo para ingresar un número de teléfono.
  • file: define un campo en una ubicación para seleccionar y subir un archivo
  • button: activa código JavaScript cuando se hace clic en él, sirve para crear un botón con un valor dado
  • checkbox: permite elegir varias opciones
  • radio: permite elegir una opción entre varias posibles.
  • range: define un control de rango (como un control deslizante)
  • hidden: sirve para ocultar el campo, para trabajar con el mismo desde el back-end.
  • reset: sirve para restablecer los campos del formulario y va a acompañado del atributo value.
  • submit: el botón para enviar el formulario

Las casillas de verificación son similares a los botones de opción, pero con una distinción: los botones opción están diseñados para seleccionar un valor de un conjunto, mientras que las casillas de verificación periten activar y desactivar valores individuales. Cuando existen varios controles, los botones de opción permiten seleccionar uno, mientras que las casillas de verificacción permiten seleccionar valores varios.

Botones

Los botones que vemos en los formularios también son un input. Se crean configurando el atributo type con el valor “submit” o “reset”.

Un input del tipo “submit” permite, al ser pulsado, que el formulario envíe los datos de sus campos a la dirección determinada por el atributo target de la etiqueta form.
Un input del tipo “reset” borra el contenido de todos los campos del formulario al ser pulsado.

Las etiquetas select u option

Las listas de opciones desplegables crean una lista de opciones, cada una contenida dentro de un elemento < option >.
Se crean como un input con el atributo type fijado en “select”, o a partir de la etiqueta select u option.
Pueden contener distintos atributos:

  • size: indica el número de valores mostrados a la vez en la lista.
  • multiple permite la selección simultánea de varios elementos de la lista.
  • id, value y name se comportan igual que en los otros campos.
  • selected: no toma ningún valor, permite que la opción que lo presenta esté elegida por defecto y sea la opción inicial.

Atributos opcionales

En muchos de los campos definidos en un formulario podemos añadir otros atributos que modifican su comportamiento:

  • required: el formulario no se podrá enviar hasta que ese campo tenga contenido.
  • placeholder: si queremos que aparezca un texto de ayuda para rellenar el campo, por ejemplo cuando le pedimos al usuario que ingrese un correo.
  • value: para introducir un valor por defecto en el campo
  • readonly: si queremos que el campo sea de sólo lectura.
