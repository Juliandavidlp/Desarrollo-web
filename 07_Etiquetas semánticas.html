Etiquetas semánticas

En versiones anteriores a HTML5, al crear la estructura de una página, normalmente se utilizaban etiquetas < div> para ir agrupando secciones de la página.
En HTML4 se utilizaban etiquetas div diferenciadas por clases. Por ejemplo:


< div class="header">Cabecera< /div>
< div class="nav">Menú de navegación< /div>

Actualmente, el elemento < div > separa áreas de un documento en divisiones


¿Qué son las etiquetas semánticas?

Las etiquetas semánticas en HTML5 nos permiten describir el contenido de una página web con un vocabulario más preciso. Son etiquetas dedicadas para cierto tipo de contenido.
Describen su significado tanto para el navegador como para el desarrollador.

Debemos respetarlas porque ayudan al navegador a entender su significado para mostrarlo en pantalla y ayudan a los buscadores a reconocer el contenido y la estructura de la página.

  • < article>: El artículo. La parte principal de un escrito (posts en blogs, artículos en diarios, mensajes en foros, comentarios, etc.)
  • < section>: Define una sección en un documento
    • .
  • < header>:Se coloca en el body y es la cabecera visual de la página o de una sección (logotipo, título, etc.). No confundir con , que es el encabezado del documento HTML.
  • < footer>: Pie de página (del documento completo) o de una sección.
  • < small>: Una aclaración cortita.
  • < nav>: Es el apartado de navegación (enlaces de secciones, categorías, etc...). También permite dividir en categorías una sección.
  • < main>:El contenido principal del body.
  • < aside>: Es una agrupación de contenido relacionado con el tema principal. Suele usarse para agregar publicidad, notas o para envolver elementos de navegación.
  • < figure>: Un elemento contenido que refiere contenido del elemento main, principal. Puede contener gráficos, fotos y pedazos de código.
  • < figcaption>: El título o el subtítulo del elemento figure
  • < adress>: Agrupación con la información de contacto del autor del artículo o documento.
